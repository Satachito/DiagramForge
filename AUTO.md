# CDE ファイル設計書（`.cde`）

このリポジトリの編集アプリが読み書きする **`*.cde` は JSON のオブジェクト**です。ブラウザ／ランタイム側でのモデルは `Web/Application.js` に準じています。

## 概要

| 項目 | 説明 |
|------|------|
| **MIME／実装詳細** | 単なる UTF-8 のテキスト JSON |
| **ルート** | `{ "nodes": [...], "links": [...] }` |
| **保存単位** | 画面上はドラフト状態がありますが、**ファイルへの Save は `nodes` と `links` のみ**（選択・ドラフトは保存しない） |

ランタイムでは選択状態として **`app.deforms`** が `[ ID, area_data, path_operation ]` と同一構造を並べますが、それは `.cde` に書き出ません。

---

## ルートオブジェクト

```json
{
	"nodes": [ … ],
	"links": [ … ]
}
```

- **`nodes`**  
  順序付きの配列。**一意なノード ID** がリンクの両端および編集対象として参照されます。
- **`links`**  
  **`nodes`** に存在する ID で結ぶ「エッジ」リスト。読み込み後に両端ノードが存在しないリンクは落ちます。

---

## ノード：`[ ID, area_data, path_operation ]`

3 要素の組として格納します。

| index | 名前 | 型・内容 |
|-------|------|-----------|
| `0` | **`ID`** | **文字列**。ノードを一意にする識別子。**空や重複は避ける**。 |
| `1` | **`area_data`** | 形状・コンテンツ（下記）。読み込み後に既定フィールドへ正規化されます。 |
| `2` | **`path_operation`** | スタイル（下記）。未定義プロパティは削除されます。 |

読み込み処理では、`area_data` の座標フィールドとして **`cX`／`cY`（レガシー）** があれば **`cx`／`cy`** に読み替え、`div` があれば **`innerHTML`** に取り込みます。

---

## `area_data`（ノード本体）

共通して使われるフィールドです。

| フィールド | 説明 |
|-------------|------|
| **`type`** | **`"rect"` \| `"ellipse"` \| `"SVG"` \| `"PNG"`**。描画タイプを決める。 |
| **`cx`** | **中心の X**（数値）。 |
| **`cy`** | **中心の Y**（数値）。 |
| **`rH`** | **横方向「半径」**。実際の幅はおよそ **`2 × \|rH\|`** に対応します。符号は描画側で絶対値などに使われます。 |
| **`rV`** | **縦方向「半径」**。実際の高さはおよそ **`2 × \|rV\|`**。 |
| **`radii`** | **省略可**。矩形の角丸。**真と評価されるとき**のみ `roundRect(..., radii)` が使われ、`0` 省略が可能。 |
| **`innerHTML`** | **省略可**。`rect` と `ellipse` で、SVG `foreignObject` の本文としてレンダリング。**`<br>` 等は XHTML に直してからシリアライズ**されます。 |
| **`style`** | **省略可**。ノード編集では CSS フラグメント（`;prop : value` 形式）として保持し、`foreignObject` のラッパーへのスタイル適用に使われます。 |

アセットノードでは次を追加します。

| `type` | 追加フィールド |
|--------|----------------|
| **`SVG`** | **`SVG`**：SVG ソース文字列。 |
| **`PNG`** | **`PNG`**：バイナリ由来の Base64（アプリ実装どおり）。 |

作成時／検証：**幅・高さともに 5px を超える**ノードのみ許可されています（`AreaData` 適用後のサイズ）。

---

## ジオメトリとの対応（実装）

矩形の左上とサイズは次のように解釈されます。

- **`x`** = `cx − rH`
- **`y`** = `cy − rV`
- **`width`** = `rH + rH`
- **`height`** = `rV + rV`

楕円・リンクの境界判定も **`cx, cy`** と **`rH, rV`** で行います。

---

## `path_operation`（描画スタイル）

ノードのアウトライン／塗りと、リンク線の両方で使われます。許可されるキーだけが残り、それ以外や値が **`undefined`** の項目は除外されます。

対応キー一覧：

- **`fill`** — 塗り色など（Canvas にそのまま渡される）
- **`stroke`** — 線の色など
- **`yellow`** — シリアライズ対象として残るフィールド。**現状の標準描画コード（`DrawPath`）では参照されない**ことがある（AI 側プロンプト等では言及できる予約）。
- **`lineWidth`**
- **`lineCap`** — `"butt"` / `"round"` / `"square"` または空で省略
- **`lineJoin`** — `"miter"` / `"round"` / `"bevel"` または空で省略
- **`miterLimit`**
- **`lineDash`** — 配列（例：`[4, 4]`）
- **`lineDashOffset`**

---

## リンク：`[ A, B, direction, path_operation ]`

4 要素の組です。

| index | 名前 | 説明 |
|-------|------|------|
| `0` | **`A`** | **始点ノード ID**（文字列、trim される） |
| `1` | **`B`** | **終点ノード ID** |
| `2` | **`direction`** | **矢印の向き**（下表） |
| `3` | **`path_operation`** | 線のスタイル（上記と同じオブジェクト） |

### `direction`

| 値 | 意味 |
|----|------|
| **`"<>"`** | 両端に矢印 |
| **`"<"`** | A 側のみ矢印（始点） |
| **`">"`** | B 側のみ矢印（終点） |
| **`""`**（空文字） | 矢印なし（線のみ） |

リンクの形状は、両ノードの **`area_data` 中心**を結び、**各ノードの外周との交点**を端点として引かれます。

編集 UI で **方向を省略**した場合の既定は **`"<>"`** と同じ動作ですが、JSON を手で書くときは明示すると安全です。

読み込み時、**`A` と `B` が空**、または **いずれも `nodes` に存在しない**リンクは **`links` に含まれません**。

---

## 同一ノード間のリンク

`ApplyLink` は **`A` と `B` の組**で既存リンクを検索し、あれば上書き・無ければ追加します。ファイルに重複があれば、読み込み後に編集で統合される想定です。**手編集する場合は一意にする**のが無難です。

## 変更履歴メモ（読み手向け）

以前の「`[ ID, TAG, G, S]`」のような表現や **`PATH` ノードタイプ**、`div` と `rect` の分離は、この設計では **統合・廃止済み**です。古いファイルは手動または移行スクリプトで上記形へ合わせてください。
